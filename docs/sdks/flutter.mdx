Flutter is the platform of choice for Stellar wallets in emerging markets. The Grat Flutter SDK enables developers to build professional-grade mobile apps where users can transact in USDC without ever seeing a gas fee.

## Installation

Add the dependency to your `pubspec.yaml`:

```yaml
dependencies:
  grat: ^1.0.0
  stellar_flutter_sdk: ^2.5.0
```

Then run:
```bash
flutter pub get
```

## Quick Setup

```dart
import 'package:grat/grat.dart';
import 'package:stellar_flutter_sdk/stellar_flutter_sdk.dart';

final grat = Grat.testnet(); // Defaults to https://api.grat.network

// Sponsor a transaction
final result = await grat.sponsor(signedXdr);
print('Success! Hash: ${result.hash}');
```

## Client Reference

### Constructor Parameters

| Parameter | Type | Description |
| :--- | :--- | :--- |
| `relayUrl` | `String` | The URL of the Grat Relay. |
| `network` | `String` | `testnet` or `mainnet`. |
| `apiKey` | `String` | Your API key (optional for testnet). |
| `maxRetries` | `int` | Number of retries (default: 3). |
| `timeout` | `Duration` | Request timeout (default: 30 seconds). |

### Methods

- `Future<SponsorResult> sponsor(String xdr)`
- `Future<SponsorResult> sponsorContract(String xdr)`
- `Future<SimulationResult> simulate(String xdr)`
- `Future<EstimateResult> estimate(String xdr)`
- `Future<HealthStatus> status()`
- `Future<CreditBalance> credits()`

## Error Handling

The SDK uses typed exceptions for precise error handling.

```dart
try {
  await grat.sponsor(txXdr);
} on AuthenticationException {
  // Handle invalid API key
} on RateLimitException catch (e) {
  print('Rate limited. Retry after ${e.retryAfter} seconds');
} on GratException catch (e) {
  print('Error: ${e.message}');
}
```

## Mobile-Specific Tips

<AccordionGroup>
  <Accordion title="Security">
    Never bundle your `GRAT_API_KEY` in the application binary. For production apps, you should proxy requests through your own backend or use a session-based key system.
  </Accordion>
  <Accordion title="Network Reliability">
    Mobile networks can be unstable. We recommend increasing the `timeout` to 60 seconds and using a package like `connectivity_plus` to detect when the user is back online before retrying.
  </Accordion>
  <Accordion title="Secure Storage">
    Store any temporary credentials or session keys using `flutter_secure_storage` to keep them encrypted on the device.
  </Accordion>
</AccordionGroup>

<Note>
  The Grat Flutter SDK is designed to work seamlessly with the `stellar_flutter_sdk` package for transaction building and signing.
</Note>
