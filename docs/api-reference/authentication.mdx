---
title: Authentication
description: "How to authenticate your requests to the Grat Relay."
---

Grat uses API keys to secure requests on the Mainnet. Testnet relays are generally open but may apply IP-based rate limits.

### API Keys
Requests to a secure relay must include the `X-API-Key` header.

```bash
curl -H "X-API-Key: sr_live_your_key_here" ...
```

### Key Prefixes
- `sr_test_`: Used for sandbox environments.
- `sr_live_`: Used for production (Mainnet) environments.

### Rate Limits
| Environment | Auth Required | Rate Limit |
| :--- | :--- | :--- |
| **Testnet** | No | 1,000 tx/day |
| **Mainnet** | Yes | Custom (based on plan) |

> [!NOTE]
> If you are self-hosting Grat, you can configure your own authentication middleware or disable it for private VPC usage.
