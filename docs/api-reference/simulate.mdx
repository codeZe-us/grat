---
title: Simulate Transaction
api: POST /v1/simulate
description: "Perform a dry-run simulation of a Soroban transaction."
---

Simulation is a required step for all Soroban transactions. It allows you to determine the footprint (ledger entries) and the resource fees (CPU/Memory) required for execution.

### Request Body

<ParamField body="transaction" type="string" required>
  The Base64-encoded XDR of the transaction to simulate.
</ParamField>

### Response

<ResponseField name="cost" type="object">
  <Expandable title="properties">
    <ResponseField name="cpuInstructions" type="string">
      CPU instructions required for execution.
    </ResponseField>
    <ResponseField name="memoryBytes" type="string">
      RAM required for execution.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="resourceFee" type="string">
  The minimum resource fee (in stroops) required.
</ResponseField>

<ResponseField name="events" type="array">
  Diagnostic events emitted during simulation.
</ResponseField>

### SDK Example

```typescript
const sim = await grat.simulate(unsignedTx);
console.log(`Instructions: ${sim.cost.cpuInstructions}`);
```
